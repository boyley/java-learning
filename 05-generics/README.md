# 05 · 泛型 Generics ⭐⭐

> 泛型提供编译期类型安全，本质是"类型参数化"。类型擦除是 Java 泛型的核心特色（也是坑），通配符 + PECS 是集合 API 的高频考点。

## 知识点索引

| # | 知识点 | 重要度 | 一句话 |
|---|---|---|---|
| 01 | [泛型基础](01-generics-basics.md) | ⭐⭐ | 泛型类/方法/接口、编译期类型安全、`extends`/`super` 上下界 |
| 02 | [类型擦除](02-type-erasure.md) | ⭐⭐⭐ | 编译期检查、运行期擦除为原始类型、桥接方法、`new T[]` 为何不行 |
| 03 | [通配符与 PECS](03-wildcards-pecs.md) | ⭐⭐ | `?` / `? extends` / `? super`、Producer-Extends-Consumer-Super |

## 学习顺序

先掌握**泛型基础**（01）会用 → 再理解**类型擦除**（02）才懂各种限制的根因 → 最后攻克**通配符 PECS**（03）这个理解门槛。

> 泛型是纯编译期特性，运行期被擦除，不涉及独立的 JVM 内存机制。
