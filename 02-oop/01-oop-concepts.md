# 01 · 面向对象三大特性（OOP Concepts）

> 封装（隐藏细节）、继承（复用扩展）、多态（同一接口多种实现）是面向对象的三大基石。面试重要度：⭐⭐⭐ 高频。

## 📖 核心知识

### 封装（Encapsulation）

把数据（字段）和操作数据的方法包在类里，对外**隐藏内部实现**，只暴露必要接口。手段是访问修饰符：字段用 `private`，通过 `public` 的 getter/setter 受控访问。好处：降低耦合、保护数据一致性、便于维护。

```java
class Account {
    private double balance;              // 隐藏
    public double getBalance() { return balance; }
    public void deposit(double amt) {    // 受控修改，可加校验
        if (amt > 0) balance += amt;
    }
}
```

### 继承（Inheritance）

子类 `extends` 父类，复用父类字段和方法并可扩展。Java 是**单继承**（一个类只能有一个直接父类），但可实现多个接口。继承体现 "is-a" 关系。注意继承会增强耦合，破坏封装，优先考虑组合（"has-a"）。

```java
class Animal { void eat() {} }
class Dog extends Animal { void bark() {} }  // Dog is-a Animal
```

### 多态（Polymorphism）

同一个引用类型、同一个方法调用，在**运行时**根据实际对象类型执行不同实现。前提三要素：**继承/实现、重写、父类引用指向子类对象（向上转型）**。

```java
Animal a = new Dog();  // 向上转型
a.eat();               // 运行时调用 Dog 重写的 eat（动态绑定）
```

多态让代码面向抽象编程，扩展新子类时调用方无需修改（开闭原则）。原理（动态绑定、虚方法表）详见 `06-polymorphism.md`。

四大基本原则常一并被问：抽象、封装、继承、多态（有的教材把"抽象"列为第四特性）。

## 🔑 面试要点

- 封装：`private` 字段 + 公开方法，隐藏实现、保护数据。
- 继承：单继承、复用扩展、"is-a"，`extends` 一个类、`implements` 多个接口。
- 多态三要素：继承/实现 + 重写 + 向上转型，运行时动态绑定。
- 多态好处：面向抽象编程，符合开闭原则，易扩展。
- 组合优于继承：继承耦合强、破坏封装，能用组合就别滥用继承。
- 抽象有时被列为第四特性：提取共性、忽略细节。

## ❓ 高频面试题

**Q：面向对象三大特性是什么？各解决什么问题？**
A：封装——隐藏内部实现、暴露受控接口，解决数据安全与耦合；继承——子类复用扩展父类，解决代码复用；多态——同一调用运行时表现不同，解决可扩展性（面向抽象编程）。

**Q：多态的实现前提是什么？**
A：三个条件：存在继承或接口实现关系、子类重写了父类方法、用父类引用指向子类对象（向上转型）。三者缺一多态无从谈起。

**Q：为什么说"组合优于继承"？**
A：继承是强耦合，子类依赖父类实现细节，父类改动易波及子类（脆弱基类问题），且破坏封装、只能单继承。组合通过持有对象、委托调用，耦合低、更灵活，能在运行时替换实现。除非明确 is-a 关系，优先组合。

**Q：封装的好处？只写 getter/setter 算封装吗？**
A：好处是隐藏实现、保护不变式、降低耦合、便于修改。仅仅机械地为每个字段生成 getter/setter 并不是真封装——真正的封装应在 setter 里做校验、对外只暴露必要行为而非全部字段。

## ⚠️ 易错点 / 加分项

- Java 类是**单继承**，别说成多继承；多继承靠实现多接口间接实现。
- 多态只针对**方法**（且是非 static、非 private、非 final 的实例方法），成员变量不具多态性（见 `06-polymorphism.md`）。
- 加分：能把三大特性对应到设计原则——封装对应迪米特法则，继承/多态对应里氏替换与开闭原则。
- 加分：指出继承的缺点（脆弱基类、耦合、破坏封装），体现工程判断力。
