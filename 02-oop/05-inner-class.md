# 05 · 内部类（Inner Class）

> 定义在类内部的类，分成员内部类、静态内部类、局部内部类、匿名内部类四种；匿名/局部内部类捕获的局部变量必须是 final 或等效 final。面试重要度：⭐⭐ 常考。

## 📖 核心知识

### 1. 成员内部类（Member Inner Class）

定义在类里、方法外，**持有外部类实例的引用**（隐式 `Outer.this`），可访问外部类所有成员（含 private）。创建需先有外部类实例：

```java
class Outer {
    private int x = 1;
    class Inner { int get() { return x; } }   // 可访问 Outer 的 x
}
Outer.Inner in = new Outer().new Inner();     // 特殊创建语法
```

### 2. 静态内部类（Static Nested Class）

`static` 修饰，**不持有外部类实例引用**，只能访问外部类的静态成员。独立性强，创建不依赖外部实例：

```java
class Outer {
    static class Nested { }
}
Outer.Nested n = new Outer.Nested();
```

典型用途：`HashMap.Node`、`ThreadPoolExecutor.Worker`，以及静态内部类单例（懒加载、线程安全）。

### 3. 局部内部类（Local Inner Class）

定义在方法/代码块内，作用域仅限该方法，可访问外部类成员和**外围方法的局部变量（须 final 或等效 final）**。用得较少。

### 4. 匿名内部类（Anonymous Inner Class）

没有名字，声明与实例化合一，用于快速实现接口/抽象类，常见于回调、监听器。JDK8 后很多可被 Lambda 替代（但 Lambda 只能实现函数式接口）：

```java
Runnable r = new Runnable() {
    @Override public void run() { System.out.println("run"); }
};
Runnable r2 = () -> System.out.println("run");  // Lambda 简化
```

### 闭包与 final 变量捕获

局部内部类和匿名内部类会**捕获**外围方法的局部变量。由于局部变量存在栈上、方法返回即销毁，而内部类对象可能存活更久，编译器实际是把该变量**复制一份**存进内部类。为保证副本与原变量一致，Java 要求被捕获的局部变量必须是 **final 或 effectively final（等效 final，即赋值一次后不再修改）**。JDK8 起不用显式写 `final`，只要事实上没被修改即可。

```java
void method() {
    int count = 0;          // 等效 final
    Runnable r = () -> System.out.println(count);  // 捕获，OK
    // count = 1;           // 若加这行，上面会编译报错
}
```

注意：**成员变量**没有这个限制（存在堆上，通过外部类引用访问），只有**局部变量**捕获才要求 final。

## 🔑 面试要点

- 四种内部类：成员、静态、局部、匿名。
- 成员内部类持有外部实例引用，可访问外部所有成员；创建用 `outer.new Inner()`。
- 静态内部类不持有外部实例，只能访问外部静态成员，独立性强。
- 局部/匿名内部类捕获的局部变量必须 final 或等效 final。
- 原因：局部变量在栈上会随方法销毁，编译器复制副本，final 保证一致性。
- 匿名内部类没名字、只能用一次，JDK8 后函数式接口场景多被 Lambda 取代。
- 成员变量捕获无 final 限制，仅局部变量有。

## ❓ 高频面试题

**Q：内部类有哪几种？**
A：成员内部类、静态内部类、局部内部类、匿名内部类。区别主要在定义位置、是否持有外部类实例引用、能访问的成员范围。

**Q：为什么局部内部类/匿名内部类访问的局部变量必须是 final（或等效 final）？**
A：局部变量存在栈上，方法结束就销毁，而内部类实例可能比方法存活更久。编译器实际是把该变量**复制一份**保存到内部类中。为避免副本与原变量不一致造成的数据歧义，Java 强制该变量不可被修改，即 final 或等效 final。JDK8 起可省略 `final` 关键字，只要事实上没修改。

**Q：静态内部类和成员内部类的区别？**
A：成员内部类隐式持有外部类实例引用，能访问外部类所有（含实例）成员，创建依赖外部实例；静态内部类不持有外部实例引用，只能访问外部静态成员，创建独立。因为持有引用，成员内部类可能导致外部类实例无法被 GC（内存泄漏风险）。

**Q：匿名内部类和 Lambda 有什么区别？**
A：匿名内部类可实现任意接口或继承类、可有多个方法和字段、`this` 指向自身；Lambda 只能实现**函数式接口**（单抽象方法）、更简洁、`this` 指向外围类实例，且不生成额外 class 文件（用 `invokedynamic`）。

## ⚠️ 易错点 / 加分项

- 成员内部类持有外部引用，长生命周期持有短生命周期外部类时会**内存泄漏**（如 Android Handler 场景）。
- 匿名内部类里的 `this` 指向匿名类自己，Lambda 里的 `this` 指向外围类——常被拿来对比。
- 等效 final 不等于加了 final：只要"赋值一次后不再改"即可，JDK8 起自动判定。
- 加分：静态内部类实现单例（Holder 模式）——利用类加载机制保证懒加载 + 线程安全，且无锁开销。
- 加分：能说清捕获的是**值的副本**而非引用，所以捕获后原变量即使能改也不影响内部类看到的值（这也是要求 final 的深层原因）。
