# 03 · 重载 vs 重写（Overload vs Override）

> 重载=同类中同名不同参（编译期静态绑定），重写=子类改写父类方法实现（运行期动态绑定）。名字像但机制完全不同。面试重要度：⭐⭐⭐ 高频。

## 📖 核心知识

**重载（Overload）**：同一个类中，方法**名相同、参数列表不同**（个数、类型、顺序不同），与返回值、访问修饰符无关。编译期根据实参类型决定调哪个——**静态绑定 / 编译期多态**。

```java
class Calc {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }  // 重载
    int add(int a, int b, int c) { return a + b + c; } // 重载
}
```

**重写（Override）**：子类重新实现父类**继承来**的方法，方法签名（名 + 参数）必须相同。运行期根据实际对象类型决定调哪个——**动态绑定 / 运行期多态**。

```java
class Animal { Object speak() { return "..."; } }
class Dog extends Animal {
    @Override
    String speak() { return "Wang"; }  // 返回类型协变，OK
}
```

### 对比表

| 维度 | 重载 Overload | 重写 Override |
| --- | --- | --- |
| 发生位置 | 同一个类（或父子类均可） | 子类 override 父类 |
| 方法名 | 相同 | 相同 |
| 参数列表 | **必须不同** | **必须相同** |
| 返回类型 | 无所谓 | 相同或**协变**（子类型） |
| 访问修饰符 | 无所谓 | **不能更严格**（可更宽） |
| 异常（受检） | 无所谓 | 不能抛更宽的受检异常 |
| 绑定时机 | 编译期（静态） | 运行期（动态） |
| 多态类型 | 编译期多态 | 运行期多态 |

### 重写的规则（"两同两小一大"）

- **两同**：方法名相同、参数列表相同。
- **两小**：返回类型 ≤ 父类（协变）、抛出的受检异常 ≤ 父类。
- **一大**：访问权限 ≥ 父类（不能缩小，如父 `public` 子不能 `protected`）。

不能重写：`private`（子类不可见，实为新方法）、`static`（属于类，是隐藏 hiding）、`final` 方法。

## 🔑 面试要点

- 重载：同名不同参，与返回值/修饰符无关，编译期静态绑定。
- 重写：签名相同，子类改实现，运行期动态绑定。
- 重写"两同两小一大"：名参同、返回和异常更小、访问权限更大。
- 返回类型**协变**（JDK5+）：子类重写方法可返回父类返回类型的子类型。
- 重写访问权限只能放宽不能收紧，异常不能抛更宽的受检异常（可不抛或抛子类）。
- `private`/`static`/`final` 方法不能被重写。
- `@Override` 注解让编译器校验是否真的构成重写，强烈建议加。

## ❓ 高频面试题

**Q：重载和重写的区别？**
A：重载是同类中同名方法参数列表不同，编译期根据参数类型确定调用（静态绑定）；重写是子类改写父类同签名方法，运行期根据实际对象类型确定调用（动态绑定）。前者是编译期多态，后者是运行期多态。

**Q：重写对返回值、访问权限、异常有什么要求？**
A："两小一大"：返回类型相同或为其子类型（协变返回）；抛出的受检异常不能比父类更宽（可更窄或不抛）；访问修饰符不能比父类更严格（只能相同或更宽）。

**Q：方法名相同、参数相同、只有返回类型不同，算重载吗？**
A：不算，且**编译报错**。重载要求参数列表不同，返回类型不参与区分方法签名，仅返回类型不同无法构成合法的重载。

**Q：`static` 方法能被重写吗？**
A：不能，构成的是**隐藏（hiding）**而非重写。静态方法属于类、编译期静态绑定，用哪个看引用的声明类型而非实际对象，没有多态。加 `@Override` 到静态方法上会编译报错。

## ⚠️ 易错点 / 加分项

- 只改返回类型不改参数不是重载（编译错误），是高频陷阱。
- 自动装箱 + 重载易踩坑：`method(int)` 与 `method(Integer)` 同时存在时优先匹配精确类型，规则复杂，避免这样设计。
- 重写访问权限"能大不能小"记反了会被扣分——父 `protected`，子可 `public`，反之不行。
- 加分：能说清"隐藏 hiding"（static/字段）与"重写 override"的区别，前者按声明类型解析，后者按实际类型。
- 加分：协变返回类型是 JDK5 才支持的，之前重写返回类型必须完全一致。
