# 05 · final 与 static（final & static）

> `final` 表"不可变/不可改"，`static` 表"属于类而非实例"；二者组合是常量。类初始化顺序（静态→实例→构造）是高频追问。面试重要度：⭐⭐ 常考。

## 📖 核心知识

### final

`final` 用在三处，语义各不同：

- **修饰变量**：一旦赋值不可再改。基本类型是值不可变；引用类型是**引用不可变**（不能指向新对象，但对象内容可改）。局部/成员/参数均可。
- **修饰方法**：不能被子类**重写**（override），可被继承和重载。
- **修饰类**：不能被**继承**，如 `String`、`Integer` 都是 final 类。

```java
final int x = 1;            // x 不可再赋值
final List<String> list = new ArrayList<>();
list.add("a");             // OK，改内容
// list = new ArrayList<>(); // 编译错误，改指向
```

成员 `final` 变量必须在**声明时 / 构造器 / 实例代码块**中完成赋值（空白 final）。

### static

`static` 表示成员属于**类**，被所有实例共享，随类加载而初始化，通过类名直接访问：

- **静态变量**：类变量，全类唯一一份，存于方法区（JDK8 后在元空间/堆）。
- **静态方法**：不依赖实例，**不能访问非静态成员**、不能用 `this`/`super`。
- **静态代码块**：`static { }`，类加载时执行一次，用于初始化静态资源。
- **静态内部类**：不持有外部类实例引用（详见 `../02-oop/05-inner-class.md`）。

### 初始化执行顺序

一个类首次使用时的初始化顺序（父类优先，静态只一次）：

```
父类静态变量 / 静态代码块（按声明顺序）
子类静态变量 / 静态代码块
父类实例变量 / 实例代码块 → 父类构造器
子类实例变量 / 实例代码块 → 子类构造器
```

```java
class A {
    static { System.out.println("A static"); }   // 1
    { System.out.println("A instance"); }         // 3
    A() { System.out.println("A ctor"); }         // 4
}
class B extends A {
    static { System.out.println("B static"); }   // 2
    { System.out.println("B instance"); }         // 5
    B() { System.out.println("B ctor"); }         // 6
}
// new B() 输出：A static, B static, A instance, A ctor, B instance, B ctor
```

要点：**静态部分只在类加载时执行一次**；每次 `new` 都执行实例代码块 + 构造器。

## 🔑 面试要点

- `final` 变量=值/引用不可变，方法=不可重写，类=不可继承。
- `final` 引用类型只锁指向，不锁内容。
- `static` 成员属于类、全局唯一、类名访问、随类加载初始化。
- 静态方法不能访问实例成员，不能用 `this`/`super`。
- 初始化顺序：父静态 → 子静态 → 父实例/构造 → 子实例/构造；静态只一次。
- 静态代码块常用于加载驱动、初始化常量表等一次性工作。
- `static final` 基本类型/字符串常量是编译期常量，会被内联到使用处。

## ❓ 高频面试题

**Q：`final` 修饰引用类型变量，对象内容能改吗？**
A：能。`final` 只保证引用不能指向别的对象，对象自身状态可变。如 `final List` 不能重新赋值，但可以 `add`/`remove` 元素。

**Q：`static` 方法里能不能用 `this`？能不能调用非静态方法？**
A：都不能。静态方法属于类、没有实例上下文，因此没有 `this`，也不能直接访问非静态成员，只能操作静态成员或先 `new` 一个实例再访问。

**Q：类的初始化顺序是怎样的？**
A：父类静态 → 子类静态（只一次）→ 父类实例代码块/构造器 → 子类实例代码块/构造器。静态随类加载执行一次，实例部分每次 `new` 都执行。

**Q：`static` 变量和实例变量存在哪、区别是什么？**
A：静态变量属于类，全类共享一份，随类加载初始化，生命周期同类；实例变量属于每个对象，各存一份，随对象创建/回收。JDK8 起静态变量随 Class 对象存于堆中（类元信息在元空间）。

## ⚠️ 易错点 / 加分项

- `final` 集合"不可变"是误区——只是引用不可变，真要内容不可变用 `Collections.unmodifiableList` 或 `List.of`。
- 静态方法不能被重写（只能被隐藏 hiding），且没有多态；`static` 方法用对象调用是坏习惯。
- 静态代码块只执行一次，别以为每次 new 都跑。
- 加分：`static final` 编译期常量被内联，改了常量值不重新编译引用方会读到旧值（需重新编译）。
- 加分：能提"空白 final"——成员 final 可延迟到构造器赋值，实现"每个实例不同但一旦定死"的常量。
